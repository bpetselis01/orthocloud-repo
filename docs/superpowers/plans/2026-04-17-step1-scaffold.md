### Task 5: Create `backend/Dockerfile`

**Files:**
- Create: `backend/Dockerfile`

- [ ] **Step 1: Create `backend/Dockerfile`**

```dockerfile
# Step 1: Base image
# Python 3.11 slim keeps the image small — no build tools included by default
# Use the exact version to ensure reproducible builds across environments
FROM python:3.11-slim

# Step 2: Set working directory
# All subsequent commands run relative to /app inside the container
WORKDIR /app

# Step 3: Install dependencies first (before copying source)
# Docker caches this layer separately — if requirements.txt hasn't changed,
# pip install is skipped on rebuild, making builds significantly faster
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Step 4: Copy application source code
# Done after pip install so source changes don't invalidate the dependency cache
COPY app/ ./app/

# Step 5: Expose the port Uvicorn will listen on
# Must match PORT in .env.example and the ports mapping in docker-compose.yml
EXPOSE 8000

# Step 6: Start the FastAPI app with Uvicorn
# --reload enables live reload when source files change (development only)
# In production this would be replaced with a gunicorn + uvicorn worker command
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--reload"]
```

### Task 10: Create `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create `.github/workflows/ci.yml`**

```yaml
name: CI

# Step 1: Trigger — run on every PR opened or updated against develop
# This ensures no broken code or linting failures reach the integration branch
on:
  pull_request:
    branches:
      - develop

jobs:
  test-and-lint:
    # Step 2: Run on a fresh Ubuntu VM
    # ubuntu-latest is the standard GitHub-hosted runner for Python projects
    runs-on: ubuntu-latest

    steps:
      # Step 3: Check out the PR branch code onto the VM
      - name: Checkout repository
        uses: actions/checkout@v4

      # Step 4: Set up the exact Python version used in the Dockerfile
      # Pinning to 3.11 ensures CI matches the container runtime exactly
      - name: Set up Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      # Step 5: Cache pip dependencies
      # Avoids re-downloading packages on every run — speeds up CI by ~60s
      - name: Cache pip dependencies
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: ${{ runner.os }}-pip-${{ hashFiles('backend/requirements.txt') }}
          restore-keys: |
            ${{ runner.os }}-pip-

      # Step 6: Install dependencies from the pinned requirements file
      - name: Install dependencies
        run: pip install -r backend/requirements.txt

      # Step 7: Run ruff linter against the backend source
      # Ruff enforces code style and catches common errors — fail fast on lint issues
      - name: Lint with ruff
        run: ruff check backend/app/

      # Step 8: Run pytest against the test suite
      # Tests run from repo root so that imports resolve correctly
      # PYTHONPATH is set so 'from app.main import app' resolves inside tests/
      - name: Run tests
        run: pytest tests/ -v
        env:
          PYTHONPATH: backend
```
